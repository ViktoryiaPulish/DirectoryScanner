# Directory Scanner

**Directory Scanner** — это WPF-утилита для анализа размера файлов и каталогов внутри выбранной папки. Сканирование выполняется в многопоточном режиме с ограничением количества одновременно работающих потоков. Результат отображается в иерархическом виде с указанием размера в байтах и процента занимаемого пространства относительно родительского каталога. Поддерживается отмена сканирования через `CancellationToken`.

---

## Возможности

- **Выбор папки** через стандартный диалог.
- **Иерархическое отображение** содержимого в `TreeView` с сортировкой по размеру (от больших к меньшим).
- **Размер и процент** для каждого элемента:
  - Файлы показывают свой абсолютный размер.
  - Папки отображают суммарный размер всех вложенных элементов и процент относительно родительской папки.
- **Иконки** 📁 для папок и 📄 для файлов.
- **Отмена сканирования** — при нажатии кнопки "Cancel" в дереве остаются уже собранные данные.
- **Многопоточное сканирование** с использованием `ThreadPool` и ограничением числа потоков через `SemaphoreSlim`.
- **Игнорирование символических ссылок** (как на файлы, так и на папки) при подсчёте размера.
- **Корректный подсчёт размеров** — размер папки вычисляется после завершения обхода всех вложенных элементов, что исключает взаимоблокировки при глубокой вложенности.
- **MVVM-архитектура** с `INotifyPropertyChanged` и `ICommand`.

---

## Структура проекта

Решение состоит из трёх проектов:

- **DirectoryScanner.Core**  
  Библиотека, содержащая модель данных (`DirectoryItem`, `ItemType`) и основной сервис сканирования (`ScannerService`). Вся логика многопоточного обхода и вычислений вынесена сюда.

- **DirectoryScanner.Tests**  
  Модульные тесты на **xUnit**, покрывающие ключевую функциональность библиотеки.

- **DirectoryScanner.Wpf**  
  WPF-приложение, реализующее пользовательский интерфейс. Использует паттерн MVVM: `MainViewModel`, конвертеры для иконок и форматирования, `RelayCommand`.

---

## Тестирование

Модульные тесты для библиотеки `DirectoryScanner.Core` реализованы с использованием **xUnit**. Они покрывают основные сценарии: корректность вычисления размеров и процентов, игнорирование символических ссылок, отмену сканирования, обработку пустых и отсутствующих папок, а также устойчивость к взаимоблокировкам при глубокой вложенности и ограниченном числе потоков. Отдельно покрыты corner-кейсы по количеству worker-ов: 1, 2 * Environment.ProcessorCount и 100.
